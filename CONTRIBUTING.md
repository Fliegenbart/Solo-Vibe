# Contributing

Solo-Vibe is in early development. We're not yet accepting code contributions, but feedback is valuable.

## How to give useful feedback

The most helpful feedback is specific and grounded:

- **"The SSH wizard section in the docs assumes I know what a VPS is"** — this helps us fix a real gap
- **"I tried to understand the deploy flow and got confused at step 3"** — this helps us improve clarity
- **"I'm a non-technical founder and this is exactly what I need"** — this validates direction

Less helpful:
- "You should add Docker support" (we know — see [roadmap](docs/roadmap.md))
- "This is like Vercel but simpler" (we're aware of the comparison — see [product overview](docs/product-overview.md))

## Ways to contribute

### Report a problem
[Open a bug report](../../issues/new?template=bug_report.md) — describe what you expected vs. what happened.

### Suggest a feature
[Open a feature request](../../issues/new?template=feature_request.md) — describe the problem you're trying to solve, not just the feature you want.

### Improve documentation
If you find unclear, incorrect, or missing documentation, open an issue describing what confused you. We'll fix it.

### Share your perspective
If you're a solo AI builder and want to share how you'd use (or not use) Solo-Vibe, open an issue tagged "feedback." Real user stories shape the product more than feature requests.

## Code contributions

We'll open code contributions once the core architecture stabilizes. If you're interested in contributing code, star the repo and watch for updates.

## Early access

Want to try Solo-Vibe when it's ready? Star this repo. We'll announce the closed beta through GitHub releases.
